Order-management Program(주문내역관리 프로그램)
(by juyoungIt)

make utility를 이용한 컴파일 방법
1. 일반모드 - make main
2. 디버깅모드 - make main_debug

<프로그램 사용방법>
프로그램을 실행하면 사용자가 이용할 수 있는 기능들이 제시
원하는 항목의 번호를 입력하고 프로그램에서 유도하는 과정을
따라서 적절한 데이터나 명령을 입력해주면 됨

<기능에 대한 설명>
1. create
주문정보를 추가
사용자가 저장할 주문정보의 요소들을 직접 타이핑하여 저장
시간값, 청구가격의 경우 시스템 내부에서 처리하여 저장

2. read
ID값, 전화번호로 조회할 수 있도록 지원
부분검색이 아닌 전체검색으로서 완전히 일치하는 값을 입력해야 조회가능

3. update
이전에 저장한 주문정보를 수정, ID값을 기준으로 탐색
시간값, ID값은 변경이 불가능하며 그 외의 값은 모두 변경가능
가격정보와 같이 정보의 수정에 따라서 갱신이 필요한 값들은 수정 후
프로그램 내부적으로 다시 계산하여 최신화

4. delete
ID값을 기준으로 삭제하도록 지원
동일한 ID코드 입력 시 해당 ID값을 가진 주문정보 삭제

5. list
저장된 주문정보를 모두 리스트의 형식으로 출력
밑에 부가적으로 저장된 데이터의 수도 같이 보여줌

6. search
저장된 주문정보를 key값을 통해 검색 및 조회
여러가지 검색옵션 지원(key값 - 시간, 제품명, ID, 주소지, 연락처, 결제수단)
시간, ID, 연락처 값의 경우 부분적인 검색도 허용함.
(사용자가 입력한 key값의 일부를 포함하는 모든 정보를 검색함.)

7. show margin
입력된 주문정보를 기반으로 발생하는 수익관련 정보를 출력

8. show report
- 프로그램상에서 저장하고 있는 정보를 보고서화 하여 파일에 저장

9. load file
파일로부터 저장된 정보를 로드

10. save file
현재 프로그램상에서 저장하고 있는 정보를 파일에 저장

11. save report
현재 프로그램상에 저장된 주문정보를 기반으로 발생하는 정보를 보고서 형식으로 파일에 저장

12. gather piece
저장된 정보를 삭제함으로서 발생하게 되는 조각들을 조각모음

13. sort 
저장된 주문정보들을 몇가지 기준에 따라서 정렬
- ID 순서(오름차순, 내림차순)
- 시간순서(오름차순, 내림차순)
